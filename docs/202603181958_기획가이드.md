# Yuno920 기획 & UI/UX 가이드

## 1. 기획 프로세스 — 무엇을 만들 것인가

### Step 1: 사이트 목적 정의

시작 전에 아래 질문에 답해보세요:

| 질문 | 예시 답변 |
|------|-----------|
| 이 사이트는 누구를 위한 것인가? | 우리 가족 (부모, 조부모 등) |
| 핵심 가치는 무엇인가? | 윤호의 성장 기록을 한곳에 모아 보관 |
| 사용자가 가장 자주 할 행동은? | 기록 작성, 사진 업로드, 과거 기록 열람 |
| 다른 서비스와 뭐가 다른가? | AI가 기록을 분석/요약, 가족만의 프라이빗 공간 |

### Step 2: 핵심 기능 목록 (Feature List)

우선순위를 P0(필수) ~ P2(나중에)로 분류합니다:

#### P0 — MVP (최소 기능 제품)
| 기능 | 설명 |
|------|------|
| 일일 기록 작성 | 날짜별로 텍스트 기록 저장 |
| 기록 목록/타임라인 | 날짜순으로 기록 열람 |
| 사용자 인증 | 가족 구성원 로그인 (Supabase Auth) |
| 기본 프로필 | 윤호 프로필 정보 (생년월일, 이름 등) |

#### P1 — 핵심 확장
| 기능 | 설명 |
|------|------|
| 사진/영상 첨부 | 기록에 사진 업로드 (Supabase Storage) |
| 카테고리/태그 | 건강, 식사, 놀이, 감정 등 분류 |
| AI 요약 | 주간/월간 기록을 AI가 요약 |
| 검색 | 키워드로 과거 기록 검색 |
| 캘린더 뷰 | 달력 형태로 기록 열람 |

#### P2 — 고도화
| 기능 | 설명 |
|------|------|
| AI 인사이트 | 성장 패턴 분석, 발달 마일스톤 추적 |
| 알림 | 기록 리마인더 (매일 저녁 알림) |
| 내보내기 | PDF/포토북 형태로 출력 |
| 가족 초대 | 링크로 가족 구성원 초대 |
| 다크모드 | 야간 사용 편의 |

### Step 3: 사용자 흐름 (User Flow)

주요 시나리오를 미리 설계합니다:

```
[첫 방문]
랜딩 페이지 → 회원가입/로그인 → 대시보드

[일일 기록]
대시보드 → "오늘 기록하기" 클릭 → 기록 작성 → 저장 → 타임라인에 반영

[과거 열람]
대시보드 → 캘린더/타임라인 → 날짜 선택 → 기록 상세 보기

[AI 기능]
대시보드 → "이번 주 요약" 클릭 → AI가 기록 분석 → 요약문 표시
```

## 3. UI/UX 표준 — 어떻게 만들 것인가

### 3-1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **따뜻함** | 가족 기록 서비스답게 차갑지 않은 톤 |
| **단순함** | 버튼 하나로 기록 시작, 복잡한 메뉴 지양 |
| **일관성** | 같은 동작은 같은 모양, 같은 위치 |
| **접근성** | 조부모도 쉽게 사용할 수 있는 글씨 크기와 대비 |
| **모바일 우선** | 대부분 폰으로 기록할 것이므로 모바일 최적화 |

### 3-2. 컬러 시스템

```
Primary:     #F97316 (따뜻한 오렌지 — 핵심 액션, 브랜드)
Secondary:   #3B82F6 (신뢰감 블루 — 보조 액션)
Background:  #FFFBF5 (밝은 크림 — 따뜻한 배경)
Surface:     #FFFFFF (카드, 입력 영역)
Text:        #1F2937 (본문 텍스트)
Text-sub:    #6B7280 (보조 텍스트)
Success:     #10B981 (성공, 완료)
Error:       #EF4444 (오류, 삭제)
Border:      #E5E7EB (구분선)
```

### 3-3. 타이포그래피

| 용도 | 크기 | 굵기 | 예시 |
|------|------|------|------|
| 페이지 제목 (H1) | 28~32px | Bold | "오늘의 기록" |
| 섹션 제목 (H2) | 22~24px | SemiBold | "최근 기록" |
| 소제목 (H3) | 18~20px | Medium | "2026년 3월 18일" |
| 본문 | 16px | Regular | 기록 내용 |
| 캡션/힌트 | 14px | Regular | 입력 플레이스홀더 |
| 라벨 | 12px | Medium | 태그, 카테고리 |

폰트 추천: **Pretendard** (한글 최적화) 또는 **Noto Sans KR**

### 3-4. 간격 (Spacing)

Tailwind CSS 기준으로 통일:

| 용도 | 값 | Tailwind |
|------|-----|----------|
| 요소 내부 패딩 | 16px | `p-4` |
| 카드 간 간격 | 16px | `gap-4` |
| 섹션 간 간격 | 32~48px | `mt-8` ~ `mt-12` |
| 페이지 좌우 패딩 | 16~24px | `px-4` ~ `px-6` |
| 버튼 내부 패딩 | 12px 24px | `py-3 px-6` |

### 3-5. 컴포넌트 표준

| 컴포넌트 | 규격 |
|----------|------|
| **버튼 (Primary)** | 높이 48px, 라운드 12px, 배경 Primary 컬러 |
| **버튼 (Secondary)** | 높이 48px, 라운드 12px, 테두리 스타일 |
| **입력 필드** | 높이 48px, 라운드 8px, 테두리 Border 컬러 |
| **카드** | 라운드 16px, 그림자 sm, 배경 Surface |
| **아바타** | 40px 원형 (헤더), 64px 원형 (프로필) |
| **네비게이션** | 하단 탭바 (모바일), 상단 바 (데스크탑) |

### 3-6. 반응형 기준 (Breakpoints)

| 기기 | 너비 | Tailwind |
|------|------|----------|
| 모바일 | ~639px | 기본값 |
| 태블릿 | 640~1023px | `sm:`, `md:` |
| 데스크탑 | 1024px~ | `lg:`, `xl:` |

**모바일 우선 설계** → 모바일 먼저 만들고 데스크탑으로 확장

## 4. 데이터베이스 설계 (Supabase)

### 테이블 구조 (초안)

```sql
-- 사용자
users
├── id (uuid, PK)
├── email
├── name
├── avatar_url
└── created_at

-- 아이 프로필
children
├── id (uuid, PK)
├── user_id (FK → users)
├── name
├── birth_date
├── avatar_url
└── created_at

-- 기록
records
├── id (uuid, PK)
├── user_id (FK → users)
├── child_id (FK → children)
├── content (text)
├── category (enum: health, meal, play, emotion, milestone, etc.)
├── mood (enum: happy, neutral, sad, sick)
├── photos (text[] — Storage URLs)
├── recorded_at (date)
├── created_at
└── updated_at

-- AI 요약
ai_summaries
├── id (uuid, PK)
├── child_id (FK → children)
├── period_type (enum: weekly, monthly)
├── period_start (date)
├── period_end (date)
├── summary (text)
└── created_at
```
