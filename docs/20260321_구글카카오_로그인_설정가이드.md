# 구글·카카오 로그인 설정 가이드

> 프론트엔드 코드는 준비됨. Supabase + OAuth 콘솔 설정만 진행하면 됨.

---

## 전체 진행 순서

```
1. 구글 클라우드 콘솔 → OAuth 클라이언트 생성
2. Supabase 대시보드 → 구글 Provider 설정
3. 카카오 개발자 콘솔 → 앱 생성, REST API 키 발급
4. Supabase 대시보드 → 카카오 Provider 설정
5. 로컬/배포 URL Redirect 허용 확인
```

---

## 1단계: 구글 로그인 설정

### 1-1. Google Cloud Console

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. **프로젝트 선택** 또는 새 프로젝트 생성
3. **APIs & Services** → **OAuth consent screen**
   - User Type: **External** (일반 사용자용)
   - 앱 이름, 지원 이메일 등 입력
   - **저장 후 계속**
4. **APIs & Services** → **Credentials** → **Create Credentials** → **OAuth client ID**
   - Application type: **Web application**
   - Name: `Yuno920` 등
   - **Authorized redirect URIs** 추가:
     ```
     https://<YOUR_SUPABASE_PROJECT_REF>.supabase.co/auth/v1/callback
     ```
   - Supabase 프로젝트 URL에서 `<YOUR_SUPABASE_PROJECT_REF>` 확인  
     예: `abcdefgh` → `https://abcdefgh.supabase.co/auth/v1/callback`
5. **Client ID**, **Client Secret** 복사

### 1-2. Supabase에서 구글 설정

1. [Supabase Dashboard](https://supabase.com/dashboard) → 프로젝트 선택
2. **Authentication** → **Providers** → **Google** 활성화
3. **Client ID**, **Client Secret** 붙여넣기
4. **Save**

---

## 2단계: 카카오 로그인 설정

### 2-1. 카카오 개발자 콘솔

1. [Kakao Developers](https://developers.kakao.com/) 로그인
2. **내 애플리케이션** → **애플리케이션 추가하기**
3. 앱 이름 입력 후 저장
4. 생성된 앱 클릭 → **앱 키**에서 **REST API 키** 복사
5. **플랫폼** → **Web** 추가
   - **사이트 도메인** 추가:
     - 개발: `http://localhost:3000`
     - 배포: `https://yuno920.vercel.app`, `https://yuno920.com`
6. **카카오 로그인** → **활성화 설정** ON
7. **Redirect URI** 설정:
   ```
   https://<YOUR_SUPABASE_PROJECT_REF>.supabase.co/auth/v1/callback
   ```
   - Supabase 콜백 URL과 동일하게 입력
8. **동의항목** → 이메일, 프로필 등 필요한 항목 설정

### 2-2. Supabase에서 카카오 설정

1. Supabase Dashboard → **Authentication** → **Providers** → **Kakao** 활성화
2. **Client ID**: 카카오 **REST API 키** 붙여넣기
3. **Client Secret**:
   - 카카오 개발자 콘솔에서 **카카오 로그인** → **보안** 설정
   - **Client Secret** 사용 여부 ON 후 발급 (선택 사항)
   - Supabase Kakao Provider가 Secret을 요구하면 발급 후 입력
4. **Save**

---

## 3단계: Redirect URL 확인

### Supabase Auth URL 설정

1. Supabase Dashboard → **Authentication** → **URL Configuration**
2. **Site URL**: `https://yuno920.com` (또는 실제 프로덕션 도메인)
3. **Redirect URLs**에 다음 추가:
   ```
   https://yuno920.com/auth/callback
   https://yuno920.vercel.app/auth/callback
   http://localhost:3000/auth/callback
   ```

---

## 4단계: 테스트

1. 로컬 실행: `cd frontend && npm run dev`
2. `/auth/login` 접속
3. **Google로 로그인** 클릭 → 구글 동의 화면 → 로그인
4. **카카오로 로그인** 클릭 → 카카오 동의 화면 → 로그인
5. 정상 시 `/dashboard` 또는 `/onboarding`으로 리다이렉트 확인

---

## 트러블슷팅

| 증상 | 확인 사항 |
|------|----------|
| 구글 로그인 후 에러 | Redirect URI가 Supabase 콜백 URL과 정확히 일치하는지 확인 |
| 카카오 로그인 후 에러 | Redirect URI, REST API 키, 도메인 허용 확인 |
| 400 Bad Request | Site URL / Redirect URLs에 현재 접속 URL 포함 여부 확인 |
| "Invalid redirect" | Supabase Auth URL Configuration 재확인 |

---

## 참고

- Supabase 프로젝트 URL: `https://<project-ref>.supabase.co`
- Supabase Auth 콜백 경로: `.../auth/v1/callback` (Supabase가 제공하는 고정 경로)
- 앱의 `/auth/callback`은 Supabase 콜백 후 최종 리다이렉트할 경로
