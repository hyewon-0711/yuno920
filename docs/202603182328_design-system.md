# Yuno920 디자인 시스템 v3

> AI 기반 아이 성장·육아 통합 플랫폼 — 모바일 우선 디자인
>
> v3 변경사항: 스페이싱 토큰 누락 수정, 접근성 명세 추가, 컴포넌트 상태(empty/loading/error) 정의, 헤더·하단탭·모달 실앱 규격 확장, CSS Variables vs Modules 경계 규칙, 다크모드 전략 명시

## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **따뜻함** | 육아·성장 서비스답게 부드럽고 따뜻한 톤 |
| **단순함** | 한 손으로 기록 가능, 복잡한 메뉴 최소화 |

| **모바일 우선** | 모든 화면을 375px 기준으로 먼저 설계 |

## 9. 컴포넌트 상태 정의 (도메인)

모든 도메인 컴포넌트는 정상/빈/로딩/에러 4가지 상태를 고려합니다.

### 9-1. 상태별 공통 패턴

| 상태 | UI 패턴 |
|------|---------|
| **정상** | 데이터 표시 |
| **빈 (Empty)** | 아이콘 + 안내 문구 + CTA 버튼 |
| **로딩 (Loading)** | 스켈레톤 shimmer 애니메이션 |
| **에러 (Error)** | 에러 아이콘 + 메시지 + 재시도 버튼 |

### 9-2. 컴포넌트별 상태 메시지

| 컴포넌트 | Empty 메시지 | Error 메시지 | CTA |

| Schedule Card | "오늘 등록된 일정이 없어요" | "일정을 불러오지 못했어요" | "+ 일정 추가" / "다시 시도" |
| Metric Card | "아직 데이터가 없어요" | "데이터를 불러오지 못했어요" | "+ 입력" / "다시 시도" |
| AI Insight Card | "기록이 부족해요. 기록을 더 남기면 AI 요약이 풍성해져요" | "AI 요약을 생성하지 못했어요" | "기록하러 가기" / "다시 시도" |

- 일반 텍스트 (< 18px bold / < 24px regular): **4.5:1** 이상
- 대형 텍스트 (≥ 18px bold / ≥ 24px regular): **3:1** 이상

## 13. CSS Architecture — Variables vs Modules 경계

### 13-1. 역할 분담

| 관리 대상 | 방식 | 위치 |
|----------|------|------|
| 컬러/스페이싱/라디우스/그림자/포커스/트랜지션 토큰 | CSS Variables (`:root`) | `globals.css` |
| 기본 리셋/폰트/body 스타일 | 글로벌 CSS | `globals.css` |
| 버튼 구조/상태 | CSS Modules | `Button.module.css` |
| 카드 구조/상태 | CSS Modules | `Card.module.css` |
| 인풋 구조/상태 | CSS Modules | `Input.module.css` |
| 네비게이션 | CSS Modules | `BottomNav.module.css` |
| 헤더 | CSS Modules | `Header.module.css` |
| 도메인 컴포넌트 | CSS Modules | 각 `*.module.css` |
| 페이지 레이아웃 | CSS Modules | 각 `page.module.css` |

### 13-2. 원칙

- **토큰(값)은 CSS Variables**, **구조(레이아웃/상태)는 CSS Modules**
- CSS Modules 내에서 `var(--brand-primary)` 같은 토큰을 참조하여 일관성 유지
- 컴포넌트 간 공유되는 값은 반드시 토큰으로 정의
- 한 컴포넌트 내에서만 쓰이는 값은 module 내 로컬 변수 허용
