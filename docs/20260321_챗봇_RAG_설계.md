# 챗봇 (RAG + LLM) 설계

> 윤호(아이) 데이터 기반, 사이트 정보 참조 도우미 챗봇

---

## 1. 목표

- **역할**: 윤호에 대한 정보와 사이트 데이터를 참조해 질문에 답하는 AI 도우미
- **예시**: "오늘 윤호 일정 브리핑해줘" → "2026년 3월 21일 토요일, 오늘은 CMS와 K-pop 댄스가 있습니다."

---

## 2. 데이터 소스

| 소스 | 테이블/API | 용도 |
|------|-----------|------|
| 아이 프로필 | `children` | 이름, 생년월일 등 기본 정보 |
| 오늘 일정 | `schedules` | "오늘 일정", "일정 브리핑" 질문 |
| 일상 기록 | `records` | 최근 기록, 감정, 활동 |
| 독서 기록 | `reading_logs` | 읽은 책, 시간 |
| 성장 지표 | `growth_metrics`, `hexagon_scores` | 성장 관련 질문 |
| 커스텀 문서 | (신규) `rag_documents` | 윤호 관련 업로드 문서 |

---

## 3. 아키텍처 (Phase 1 → Phase 2)

### Phase 1: 컨텍스트 기반 채팅 (현재 구현 가능)

```
[사용자 질문] → [API: 일정/기록/독서 조회] → [프롬프트에 컨텍스트 삽입] → [LLM] → [응답]
```

- DB에서 관련 데이터 실시간 조회
- 조회 결과를 프롬프트에 포함
- 별도 벡터 DB 없음

### Phase 2: RAG (Retrieval-Augmented Generation)

```
[업로드 문서] → [Chunk 분할] → [임베딩] → [벡터 DB 저장]
[사용자 질문] → [질문 임베딩] → [벡터 유사도 검색] → [관련 Chunk + DB 조회] → [LLM] → [응답]
```

- **임베딩**: OpenAI `text-embedding-3-small`
- **벡터 DB**: Supabase pgvector 또는 Pinecone
- **Chunk**: 500~1000자, 오버랩 50~100자

---

## 4. 커스텀 문서 업로드/관리 (Phase 2)

| 기능 | 설명 |
|------|------|
| 업로드 | PDF/텍스트/이미지 → 텍스트 추출 → Chunk → 임베딩 → 저장 |
| 관리 | 문서 목록, 삭제, 재색인 |
| 스키마 | `rag_documents`, `rag_chunks`, `rag_embeddings` |

---

## 5. 프롬프트 구조 (Phase 1)

```
시스템: 당신은 윤호의 일상을 돕는 AI 도우미입니다.
오늘 날짜: 2026-03-21 (토)
아이 정보: {child.name}, 생년 {age}세

[오늘 일정]
- 10:00 CMS
- 14:00 K-pop 댄스

[최근 기록]
- 3/20: 오늘 학교에서...

[최근 독서]
- 어린왕자, 30분

사용자: {message}

규칙: 존댓말, 150자 이내, 일정/기록을 참고해 답변.
```

---

## 6. 구현 순서

| 단계 | 작업 | 산출물 |
|------|------|--------|
| 1 | 메뉴에 챗봇 추가 | Sidebar, BottomNav |
| 2 | 챗 페이지 + 기본 UI | `/chat` |
| 3 | Context-aware Chat API | `POST /api/ai/chat-assistant` |
| 4 | 연동 및 테스트 | "오늘 일정 브리핑" 동작 |
| 5 | (Phase 2) RAG 파이프라인 | 문서 업로드, 벡터 저장, 검색 |
| 6 | (Phase 2) 쿼리 시 RAG 검색 반영 | 유사 Chunk + DB 조회 병합 |

---

## 7. API 스펙 (Phase 1)

```
POST /api/ai/chat-assistant
Request: { child_id: string, message: string }
Response: { reply: string }

내부 동작:
1. get_child(child_id)
2. get_today_schedules(child_id)
3. get_records_by_date(child_id, today)
4. get_reading_logs_by_date(child_id, today)
5. 프롬프트 구성 → OpenAI 호출 → reply 반환
```
