## 1. 인증 구조

```
┌──────────┐      Supabase Auth       ┌──────────────┐
│ Next.js  │ ◄──── JWT Token ────────► │  Supabase    │
│ Frontend │                           │  (DB/Storage)│
└────┬─────┘                           └──────────────┘
     │
     │ Authorization: Bearer <supabase_access_token>
     ▼
┌──────────┐      Supabase Service     ┌──────────────┐
│ FastAPI  │ ◄──── Role Key ──────────► │  Supabase    │
│ Backend  │                            │  (DB 조회)   │
└──────────┘                            └──────────────┘
```

- **프론트엔드 → Supabase**: `supabase-js` 클라이언트로 직접 CRUD 수행 (RLS 적용)
- **프론트엔드 → FastAPI**: Supabase access token을 `Authorization` 헤더에 포함
- **FastAPI → Supabase**: Service Role Key로 DB 조회 (RLS 우회), 사용자 인증은 JWT 검증으로 수행

## 2. Supabase Client API

프론트엔드에서 `@supabase/supabase-js`로 직접 호출하는 API. RLS 정책에 의해 자동으로 접근 제어된다.

### 2-1. Auth (인증)

#### 이메일 회원가입

```typescript
const { data, error } = await supabase.auth.signUp({
  email: string,
  password: string,
  options: {
    data: { name: string }
  }
})
```

| 항목 | 설명 |
|------|------|
| 트리거 | `handle_new_user()` → `public.users` 자동 생성 |
| 성공 | `data.user` 반환, 이메일 확인 메일 발송 |
| 실패 | `error.message`: "User already registered" 등 |

#### 이메일 로그인

```typescript
const { data, error } = await supabase.auth.signInWithPassword({
  email: string,
  password: string
})
```

#### 소셜 로그인 (Google / Kakao)

```typescript
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'google' | 'kakao',
  options: {
    redirectTo: `${window.location.origin}/auth/callback`
  }
})
```

#### 로그아웃

```typescript
const { error } = await supabase.auth.signOut()
```

#### 세션 확인

```typescript
const { data: { session } } = await supabase.auth.getSession()
const { data: { user } } = await supabase.auth.getUser()
```

#### 세션 변경 리스너

```typescript
supabase.auth.onAuthStateChange((event, session) => {
  // event: 'SIGNED_IN' | 'SIGNED_OUT' | 'TOKEN_REFRESHED' | ...
})
```

### 2-2. Children (아이 프로필)

#### 아이 목록 조회

```typescript
const { data, error } = await supabase
  .from('children')
  .select('*')
  .order('created_at', { ascending: true })
```

**응답 타입**:

```typescript
interface Child {
  id: string
  user_id: string
  name: string
  birth_date: string       // "2018-09-20"
  gender: 'male' | 'female'
  birth_time: string | null // "14:30:00"
  avatar_url: string | null
  created_at: string
}
```

#### 아이 프로필 생성

```typescript
const { data, error } = await supabase
  .from('children')
  .insert({
    name: string,          // 필수
    birth_date: string,    // 필수 "YYYY-MM-DD"
    gender: 'male' | 'female', // 필수
    birth_time?: string,   // 선택 "HH:MM"
    avatar_url?: string    // 선택
  })
  .select()
  .single()
```

| 항목 | 설명 |
|------|------|
| 트리거 | `handle_new_child()` → `family_members`에 admin 역할 자동 추가 |

#### 아이 프로필 수정

```typescript
const { data, error } = await supabase
  .from('children')
  .update({ name?, birth_date?, gender?, birth_time?, avatar_url? })
  .eq('id', childId)
  .select()
  .single()
```

### 2-3. Schedules (일정)

#### 일정 목록 조회 (날짜 범위)

```typescript
const { data, error } = await supabase
  .from('schedules')
  .select('*')
  .eq('child_id', childId)
  .gte('start_time', startDate)   // "2026-03-18T00:00:00"
  .lte('start_time', endDate)     // "2026-03-18T23:59:59"
  .order('start_time', { ascending: true })
```

**응답 타입**:

```typescript
interface Schedule {
  id: string
  child_id: string
  title: string
  location: string | null
  start_time: string    // ISO 8601
  end_time: string | null
  repeat_type: 'none' | 'daily' | 'weekly' | 'monthly'
  created_at: string
}
```

#### 일정 생성

```typescript
const { data, error } = await supabase
  .from('schedules')
  .insert({
    child_id: string,
    title: string,
    location?: string,
    start_time: string,
    end_time?: string,
    repeat_type?: 'none' | 'daily' | 'weekly' | 'monthly'
  })
  .select()
  .single()
```

#### 일정 수정

```typescript
const { data, error } = await supabase
  .from('schedules')
  .update({ title?, location?, start_time?, end_time?, repeat_type? })
  .eq('id', scheduleId)
  .select()
  .single()
```

#### 일정 삭제

```typescript
const { error } = await supabase
  .from('schedules')
  .delete()
  .eq('id', scheduleId)
```

const { data, error } = await supabase.storage

### 3-1. AI 엔드포인트

#### POST /api/ai/daily-summary

하루 요약 리포트를 생성한다.

| 항목 | 값 |
|------|-----|
| Auth | 필수 |
| Rate Limit | 5회/일/아이 |

**Request Body**:

```typescript
{
  "child_id": string,    // UUID
  "date": string         // "2026-03-18"
}
```

**Response (200)**:

```typescript
{
  "success": true,
  "data": {
    "report_id": string,
    "child_id": string,
    "report_type": "daily",
    "period_start": "2026-03-18",
    "period_end": "2026-03-18",
    "content": string,      // Markdown 형식 요약문
    "metadata": {
      "mood_distribution": { "happy": 3, "neutral": 1 },
      "top_keywords": ["수학", "태권도"],
      "record_count": 4,
      "reading_count": 1,
      "total_reading_minutes": 30
    },
    "created_at": string
  }
}
```

**에러**:

| 코드 | 상태 | 설명 |
|------|------|------|
| INSUFFICIENT_DATA | 400 | 해당 날짜에 기록이 부족 (최소 1건 필요) |
| REPORT_EXISTS | 409 | 이미 생성된 리포트 존재 (캐시 반환 안내) |
| AI_GENERATION_FAILED | 502 | OpenAI API 호출 실패 |
| RATE_LIMIT_EXCEEDED | 429 | 일일 생성 횟수 초과 |

## 4. 에러 코드 정의

### HTTP 상태 코드

| 코드 | 의미 | 사용 |
|------|------|------|
| 200 | 성공 | 정상 응답 |
| 400 | Bad Request | 잘못된 요청, 데이터 부족 |
| 401 | Unauthorized | 인증 토큰 누락/만료 |
| 403 | Forbidden | 권한 없음 |
| 404 | Not Found | 리소스 없음 |
| 409 | Conflict | 중복 리소스 |
| 429 | Too Many Requests | Rate Limit 초과 |
| 500 | Internal Server Error | 서버 내부 오류 |
| 502 | Bad Gateway | 외부 서비스 (OpenAI, 기상청 등) 호출 실패 |

### 비즈니스 에러 코드

| 코드 | HTTP | 설명 |
|------|------|------|

| CHILD_NOT_FOUND | 404 | 아이 프로필을 찾을 수 없음 |

| AI_GENERATION_FAILED | 502 | AI 생성 실패 |
| AI_TIMEOUT | 504 | AI 응답 시간 초과 (30초) |
| RATE_LIMIT_EXCEEDED | 429 | Rate Limit 초과 |
| INVALID_COORDINATES | 400 | 좌표값 오류 |
| SCHOOL_NOT_FOUND | 404 | 학교 정보 없음 |
| MEAL_NOT_AVAILABLE | 404 | 급식 데이터 없음 |
| WEATHER_API_FAILED | 502 | 날씨 API 실패 |

## 5. Rate Limiting 정책

| 엔드포인트 | 제한 | 단위 | 기준 |
|-----------|------|------|------|

| /api/ai/weekly-report | 3회 | 주 | child_id |
| /api/ai/monthly-report | 2회 | 월 | child_id |
| /api/ai/coaching | 10회 | 일 | child_id |
| /api/ai/personality | 3회 | 월 | child_id |
| /api/ai/hexagon | 10회 | 일 | child_id |
| /api/ai/growth-advice | 5회 | 일 | child_id |
| /api/ai/auto-tag | 30회 | 일 | user_id |
| /api/external/weather | 30회 | 시간 | user_id |
| /api/external/meal | 30회 | 시간 | user_id |
| /api/play/generate-quiz | 10회 | 일 | child_id |

### 구현 방식

- Redis 기반 Sliding Window Counter
- 응답 헤더에 남은 횟수 포함:

```
X-RateLimit-Limit: 5
X-RateLimit-Remaining: 3
X-RateLimit-Reset: 1711036800
```

|--------|------|
| Storage 업로드 | 파일당 최대 10MB |
| 사진 첨부 | 기록당 최대 5장 |
| Realtime 채널 | 사용자당 최대 5개 |
