D-1. Accessibility

WCAG 2.1 AA 기준. 일반 텍스트 4.5:1, 대형 텍스트(18px bold / 24px regular 이상) 3:1 대비율 충족.
조합대비율AA (일반)AA (대형)
--text-primary (#111827) on --bg-card (#FFF)17.3:1✅ Pass✅ Pass
--text-secondary (#374151) on --bg-card (#FFF)10.7:1✅ Pass✅ Pass
--text-tertiary (#6B7280) on --bg-card (#FFF)5.0:1✅ Pass✅ Pass
--text-disabled (#9CA3AF) on --bg-card (#FFF)2.9:1❌ Fail✅ Pass
--brand-primary (#F97316) on --bg-card (#FFF)3.3:1❌ Fail✅ Pass
--text-inverse (#FFF) on --action-primary (#F97316)3.3:1⚠️ Large only✅ Pass
결정사항: --text-disabled는 비활성 UI에만 사용 (WCAG 예외). Primary 버튼 텍스트는 16px semibold로 대형 텍스트 기준 충족. 소형 텍스트에 Primary 색상 단독 사용 시 --brand-primary-dark (#EA580C) 사용 권장 (4.6:1).

D-2. CSS Architecture — Variables vs Modules

어디까지 CSS Variables(글로벌 토큰)로 관리하고, 어디까지 CSS Modules(컴포넌트 스코프)에서 처리할지 규칙.
관리 대상방식위치
컬러 토큰CSS Variables (:root)globals.css
스페이싱 토큰CSS Variables (:root)globals.css
라디우스 토큰CSS Variables (:root)globals.css
그림자 토큰CSS Variables (:root)globals.css
포커스 링 토큰CSS Variables (:root)globals.css
트랜지션 토큰CSS Variables (:root)globals.css
타이포그래피 (기본)CSS Variables + 유틸globals.css
버튼 스타일CSS ModulesButton.module.css
카드 스타일CSS ModulesCard.module.css
인풋 스타일CSS ModulesInput.module.css
네비게이션CSS ModulesBottomNav.module.css
도메인 컴포넌트CSS Modules각 컴포넌트.module.css
페이지 레이아웃CSS Modules각 page.module.css
원칙: 토큰(값)은 CSS Variables, 구조(레이아웃/상태)는 CSS Modules.
CSS Modules 내에서 var(--brand-primary) 같은 토큰을 참조하여 일관성 유지.

D-3. Theme Strategy

현재: Light 테마 전용으로 운영
P2 확장: [data-theme="dark"] 셀렉터로 :root 토큰 값 오버라이드
구현 방식: body 또는 html에 data-theme 속성 토글 → CSS Variables 자동 전환
컴포넌트 영향: 토큰 참조로 작성된 CSS Modules는 수정 없이 다크모드 대응
