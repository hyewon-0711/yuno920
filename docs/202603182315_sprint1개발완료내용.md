## 1. AUTH-01: 로그인 / 회원가입

### 구현 내용

**로그인 페이지 (`/auth/login`)**
- 이메일 + 비밀번호 로그인 (Supabase Auth)
- Google 소셜 로그인 버튼 (Google 로고 SVG 포함)
- 카카오 소셜 로그인 버튼 (카카오 노란색 테마)
- 비밀번호 찾기 링크
- 에러 처리: 잘못된 자격 증명 시 메시지 표시
- 로그인 성공 시 `/dashboard`로 리다이렉트

**회원가입 페이지 (`/auth/signup`)**
- 이름 + 이메일 + 비밀번호 + 비밀번호 확인 폼
- 프론트엔드 유효성 검사:
  - 이름: 2자 이상
  - 비밀번호: 8자 이상, 영문+숫자 포함
  - 비밀번호 확인 일치
- 이미 등록된 이메일 에러 처리
- Google/카카오 간편 가입 버튼
- 가입 성공 시 `/onboarding`으로 리다이렉트

**UI 특징**
- "간편 로그인" / "간편 가입" 구분선(divider) 포함
- Google: 흰색 배경 + 4색 로고
- 카카오: 노란색(#FEE500) 배경 + 카카오 로고

### 파일

| 파일 | 역할 |
|------|------|
| `app/auth/login/page.tsx` | 로그인 페이지 |
| `app/auth/login/page.module.css` | 로그인/회원가입 공통 스타일 |
| `app/auth/signup/page.tsx` | 회원가입 페이지 |

## 4. HOME-01: 오늘 일정 CRUD

### 기능 상세

- **조회**: 오늘(00:00~23:59) 범위의 일정을 시간순 정렬
- **추가**: 제목 + 시간 입력 폼 (인라인)
- **삭제**: 각 일정 옆 ✕ 버튼
- **상태 표시**:
  - `completed`: 종료된 일정 → 회색 + 취소선
  - `ongoing`: 현재 진행 중 → Primary 컬러 "진행 중" 뱃지
  - `upcoming`: 예정된 일정 → "N시간 후" / "N분 후" 뱃지
- **빈 상태**: "오늘 등록된 일정이 없습니다"

### Supabase 연동

```
schedules 테이블
├── SELECT: child_id + 오늘 날짜 범위 필터
├── INSERT: child_id, title, start_time, repeat_type
└── DELETE: id 기반
```
