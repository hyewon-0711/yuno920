# Yuno920 사용자 흐름도 v2 (User Flow)

> v2 변경사항: Actor 정의, P0 범위 축소, AI 생성 정책, 예외 흐름, 데이터 책임표, Quick Record 추가

---

## Actor 정의

| Actor | 설명 | 권한 |
|-------|------|------|
| **Parent** | 주 사용자 (엄마/아빠). 기록, 일정, 성장, AI 리포트 관리 | 모든 기능 접근 |
| **Child** | 아이. 놀이 중심 사용 | Play, 일부 기록 입력 |
| **Family Admin** | 가족 그룹 관리자 (최초 가입한 Parent) | 가족 초대, 권한 관리 |
| **Viewer** | 초대된 가족 구성원 (조부모 등) | 읽기 전용 |

## 1. 첫 사용 플로우 (온보딩) — Actor: Parent

```
[앱 진입]
   │
   ▼
[로그인 여부 확인] ──── 로그인됨 ──→ [Home (Today)]
   │
   로그인 안됨
   │
   ▼
[랜딩 페이지]
   │
   ├── "시작하기" ──→ [회원가입]
   │                      │
   │                      ▼
   │                 [이메일/비밀번호 입력]
   │                      │
   │                      ▼ (성공)
   │                 [아이 프로필 등록]
   │                   - 이름 (필수)
   │                   - 생년월일 (필수)
   │                   - 성별 (필수)
   │                   - 출생 시간 (선택)
   │                   - 사진 (선택)
   │                      │
   │                      ▼
   │                 [Home (Today)] ← 첫 화면
   │
   └── "로그인" ──→ [로그인 페이지]
                       │
                       ├── 이메일 로그인
                       ├── Google 로그인
                       └── 카카오 로그인
                            │
                            ▼ (성공)
                       [Home (Today)]
```

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| 회원가입 성공 | → 아이 프로필 등록 화면 |
| 회원가입 실패 (이메일 중복) | → 에러 메시지 + 로그인 유도 |
| 로그인 성공 | → Home (Today) |
| 로그인 실패 (비밀번호 오류) | → "비밀번호가 올바르지 않습니다" + 재시도 |
| 소셜 로그인 실패 | → "로그인에 실패했습니다. 다시 시도해주세요" |
| 아이 프로필 등록 스킵 불가 | → 반드시 1명 이상 등록 후 진입 |

## 2. 일일 루틴 플로우 (핵심 시나리오) — Actor: Parent

```
[앱 열기]
   │
   ▼
[Home (Today)]
   │
   ├── 오늘 일정 확인
   │     └── [+ 일정 추가] → 제목/시간 입력 → 저장 → 목록 갱신
   │
   ├── AI 코칭 확인 (읽기만)
   │     └── P0: placeholder 또는 간단 메시지
   │         P1: AI 생성 코칭
   │
   └── "오늘 기록하기" CTA 클릭
         │
         ▼
      [기록 입력 방식 선택]
         │
         ├── Quick Record (빠른 기록)
         │     ├── 기분 선택 (😊😐😢🤒😴)
         │     ├── 한 줄 메모
         │     └── 저장
         │
         └── Full Record (상세 기록)
               ├── 기분 선택
               ├── 카테고리 선택
               ├── 텍스트 입력
               ├── 사진 첨부 (선택, 최대 5장)
               └── 저장
                    │
                    ├── Supabase DB (records 테이블) 저장
                    ├── 사진 → Supabase Storage 업로드
                    └── 자동 태깅 (FastAPI → OpenAI)
                          │
                          ▼
                    [타임라인에 반영]
                    토스트: "기록이 저장되었습니다 ✅"
```

### P0 Home 구성 (최소)

| 요소 | 포함 여부 | 비고 |
|------|-----------|------|
| 오늘 일정 | P0 | 사용자 직접 입력 |
| 오늘 기록하기 CTA | P0 | 핵심 전환 버튼 |
| AI 코칭 | P0: placeholder | "오늘도 좋은 하루 보내세요" 고정 메시지 |
| 독서 현황 | P1 | 독서 기록 기능과 함께 |
| 급식 메뉴 | P1 | 외부 데이터 연동 필요 |
| 날씨 | P1 | 외부 API 연동 필요 |

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| Quick Record 저장 성공 | → 토스트 + Home으로 복귀 |
| Full Record 저장 성공 | → 토스트 + Record 탭 타임라인 |
| 사진 업로드 실패 | → "사진 업로드에 실패했습니다. 기록은 저장되었습니다" (텍스트만 저장) |
| 자동 태깅 실패 | → 기록은 저장, 태그는 미분류로 처리 (백그라운드 재시도) |
| 일정 추가 성공 | → 목록 갱신 + 토스트 |

## 3. 기록 열람 플로우 — Actor: Parent / Viewer(읽기만)

```
[Record 탭]
   │
   ▼
[기록 목록] ← 일반 기록 + 독서 기록 통합 뷰
   │
   ├── [타임라인 뷰] ← 기본
   │     - 날짜별 그룹
   │     - 기록 타입 아이콘으로 구분 (📝 일반 / 📚 독서)
   │     - 스크롤로 과거 기록 열람
   │
   └── [캘린더 뷰]
         - 달력에서 날짜 클릭
              │
              ▼
         해당 날짜의 기록 표시
   │
   기록 카드 클릭
   │
   ▼
[기록 상세]
   │
   ├── 내용 열람
   ├── [수정] → 수정 폼 → 저장 → 상세로 복귀
   └── [삭제] → 확인 팝업 → 삭제 → 목록으로 복귀
```

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| 수정 저장 성공 | → 토스트 + 기록 상세 화면 갱신 |
| 삭제 확인 | → 토스트 + 기록 목록으로 이동 |
| 삭제 취소 | → 팝업 닫기, 상세 화면 유지 |

## 4. 독서 기록 플로우 — Actor: Parent

> 저장 구조: `reading_logs` 테이블 (records와 분리)
> 표시 구조: Record 목록에서 일반 기록과 통합 뷰

```
[Record 탭 → "+ 기록" → 독서 기록 선택]
  또는
[Home → 독서 현황 카드 → "+ 독서 기록"] (P1)
   │
   ▼
[독서 기록 작성]
   ├── 책 제목 입력
   ├── 카테고리 선택 (과학/동화/역사/예술/기타)
   ├── 읽은 날짜
   ├── 읽은 시간 (분)
   └── 메모 (선택)
        │
        ▼
     [저장]
        │
        ├── reading_logs 테이블 저장
        ├── Home 독서 현황 갱신 (P1)
        ├── Growth 독서 성장 데이터 갱신 (P1)
        └── Play 독서 퀴즈 소스 반영 (P2)
```

### 데이터 구조 결정

| 항목 | 결정 |
|------|------|
| 일반 기록 | `records` 테이블 |
| 독서 기록 | `reading_logs` 테이블 (별도) |
| 목록 표시 | 통합 뷰 (타임라인에 두 타입 함께 표시) |
| 구분 방법 | 아이콘 + 카드 스타일 차이 |

## 5. 성장 확인 플로우 — Actor: Parent

```
[Growth 탭]
   │
   ▼
[성장 대시보드]
   │
   ├── 신체 성장 카드
   │     ├── 그래프 확인 (읽기)
   │     └── [+ 입력] → 키/몸무게 입력 → 저장 → 그래프 갱신
   │
   ├── 6각형 인재 분석
   │     ├── 레이더 차트 확인 (읽기)
   │     └── AI 분석 코멘트 확인
   │
   ├── 학습 지표
   │     ├── SR 점수 추이 확인
   │     └── [+ 입력] → 점수 입력 → 저장
   │
   └── 독서 성장
         ├── 월별 독서량 차트
         └── 분야별 분포 차트
```

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| 신체 데이터 입력 성공 | → 그래프 즉시 갱신 + 토스트 |
| 데이터 없음 (첫 사용) | → "아직 데이터가 없어요. 첫 기록을 입력해보세요" + 입력 CTA |
| 6각형 데이터 부족 | → 부족한 영역 표시 + "놀이/기록을 더 해보세요" 안내 |

## 6. 놀이 플로우 — Actor: Child

```
[Play 탭]
   │
   ▼
[놀이터 메인]
   │
   ├── [계산 게임] 선택
   │     │
   │     ▼
   │   [게임 시작 화면]
   │     - 현재 레벨 표시
   │     - "시작" 버튼
   │        │
   │        ▼
   │   [게임 플레이]
   │     - 문제 표시
   │     - 답 입력 (터치 키패드)
   │     - 맞으면 다음 문제
   │     - 틀리면 정답 표시
   │        │
   │        ▼ (10문제 또는 시간 종료)
   │   [결과 화면]
   │     - 점수
   │     - 정답률
   │     - 레벨 변화
   │     - "다시하기" / "돌아가기"
   │        │
   │        ▼
   │   play_logs에 저장 → hexagon_scores 갱신
   │
   ├── [기억력 게임] (유사 흐름)
   │
   └── [독서 퀴즈]
         │
         ▼
      [최근 읽은 책 기반 퀴즈]
        - 퀴즈 데이터 확인
        - 있으면: 즉시 시작
        - 없으면: 생성 요청 → 로딩 → 시작
        - 객관식 3~5문제
        - 결과 저장 → play_logs + hexagon_scores 갱신
```

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| 게임 완료 | → 결과 화면 → 놀이터 메인 또는 다시하기 |
| 독서 퀴즈 데이터 없음 (책 기록 0건) | → "먼저 책을 읽고 기록해보세요!" + Record 탭 유도 |
| 독서 퀴즈 생성 실패 | → "퀴즈를 만들지 못했어요. 다시 시도해주세요" + 재시도 버튼 |
| 게임 중 앱 이탈 | → 진행 상태 미저장, 재시작 필요 |

## 7. AI 리포트 플로우 — Actor: Parent

```
[Insight 탭]
   │
   ▼
[AI 인사이트 메인]
   │
   ├── [하루] 탭 ← 기본
   │     │
   │     ▼
   │   [하루 요약 카드]
   │     - 캐시 확인 → 있으면 즉시 표시
   │     - 없으면: "요약 생성하기" 버튼
   │          │
   │          ▼
   │        FastAPI → Supabase에서 오늘 데이터 조회
   │                → OpenAI로 요약 생성
   │                → ai_reports에 저장 (캐시)
   │                → 화면에 표시
   │
   ├── [주간] 탭
   │     - 주간 행동 패턴 분석
   │     - 변화 포인트 하이라이트
   │
   ├── [월간] 탭
   │     - 성장 분석 리포트
   │     - 독서 성향 분석
   │
   └── [기질 리포트] 카드 클릭
         │
         ▼
      [기질/사주 리포트 상세]
        - 입력: 생년월일 + 출생시간
        - 캐시 확인 → 있으면 즉시 표시
        - 없으면: 생성 → ai_reports에 저장
        - 성향 분석 + 양육 가이드 표시
        - "재생성" 버튼 별도 제공
```

### AI 생성 정책

| 리포트 종류 | 생성 시점 | 캐시 정책 | 재생성 |
|------------|----------|----------|--------|
| **하루 요약** | 사용자가 "생성하기" 버튼 클릭 시 (수동) | 당일 1회 생성 후 캐시, 다음 날 만료 | 당일 내 수동 재생성 가능 |
| **주간 리포트** | 매주 월요일 배치 생성 (P2) / 초기에는 수동 | 7일 캐시 | 수동 재생성 가능 |
| **월간 리포트** | 매월 1일 배치 생성 (P2) / 초기에는 수동 | 30일 캐시 | 수동 재생성 가능 |
| **독서 퀴즈** | Play 진입 시 생성 → 생성 후 저장 | 책 단위 캐시 (같은 책 재사용) | 새 퀴즈 요청 가능 |
| **기질 리포트** | 최초 1회 생성 후 영구 캐시 | 영구 (생년월일 변경 시 무효화) | "재생성" 버튼으로 수동 |
| **AI 코칭 (Home)** | P0: 고정 메시지 / P1: 매일 첫 접속 시 생성 | 당일 캐시 | 자동 |

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| AI 요약 생성 성공 | → 카드에 즉시 표시 |
| AI 요약 생성 실패 | → "요약을 생성하지 못했어요. 잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 기록 데이터 부족 (하루 요약) | → "오늘의 기록이 부족해요. 기록을 더 남기면 AI 요약이 더 풍성해져요" |
| 기질 리포트 출생시간 미입력 | → "출생 시간을 입력하면 더 정확한 분석이 가능해요" + 설정 이동 |
| API 비용 제한 초과 | → "오늘 생성 횟수를 초과했어요" (일일 한도 설정 필요) |

## 8. 설정 / 프로필 플로우 — Actor: Parent (Family Admin)

```
[Home 헤더 → 프로필 아이콘] 또는 [설정 메뉴]
   │
   ▼
[설정 페이지]
   │
   ├── [P0] 아이 프로필 편집
   │     └── 이름/생년월일/사진 수정 → 저장
   │
   ├── [P0] 로그아웃
   │     └── 확인 팝업 → 세션 삭제 → 랜딩 페이지
   │
   ├── [P1] 알림 설정
   │     ├── 기록 리마인더 ON/OFF
   │     ├── 리포트 알림 ON/OFF
   │     └── 알림 시간 설정
   │
   ├── [P2] 가족 관리
   │     ├── 가족 목록 확인
   │     └── [+ 가족 초대]
   │           └── 초대 링크 생성 → 공유
   │                 → 수신자가 링크 클릭 → 가입 → family_members에 추가
   │
   └── [P2] 고급 설정
         ├── 다크모드
         ├── 데이터 내보내기
         └── 계정 삭제
```

### Entry / Exit 조건

| 조건 | 결과 |
|------|------|
| 프로필 수정 성공 | → 토스트 + 설정 화면 유지 |
| 로그아웃 확인 | → 세션 삭제 → 랜딩 페이지 |
| 로그아웃 취소 | → 팝업 닫기 |
| 가족 초대 링크 생성 | → 공유 시트 (카카오/문자/복사) |
| 초대 수신자 가입 완료 | → 가족 목록에 자동 추가 |

## 예외 흐름 (Exception Flow)

### 네트워크/시스템

| 상황 | 사용자에게 보이는 것 | 시스템 처리 |
|------|---------------------|------------|
| 네트워크 끊김 | "인터넷 연결을 확인해주세요" 배너 | 오프라인 상태 감지, 재연결 시 자동 새로고침 |
| 서버 오류 (500) | "잠시 문제가 발생했어요. 다시 시도해주세요" | 에러 로그 + 재시도 버튼 제공 |
| 세션 만료 | "다시 로그인해주세요" | 자동 로그인 페이지 이동 |

### 데이터 입력

| 상황 | 사용자에게 보이는 것 | 시스템 처리 |
|------|---------------------|------------|
| 사진 업로드 실패 | "사진 업로드에 실패했습니다" | 텍스트 기록만 저장, 사진은 재업로드 유도 |
| 사진 용량 초과 | "사진 크기가 너무 큽니다 (최대 10MB)" | 업로드 차단 |
| 필수 필드 미입력 | 해당 필드 빨간 테두리 + "필수 항목입니다" | 저장 버튼 비활성 |

### AI 관련

| 상황 | 사용자에게 보이는 것 | 시스템 처리 |
|------|---------------------|------------|
| AI 요약 생성 실패 | "AI 요약을 생성하지 못했어요" + 재시도 버튼 | 에러 로그, 최대 2회 자동 재시도 후 실패 표시 |
| 독서 퀴즈 생성 실패 | "퀴즈를 만들지 못했어요" + 재시도 버튼 | 에러 로그 |
| AI 응답 지연 (>10초) | 스켈레톤 로딩 + "AI가 분석 중이에요..." | 30초 타임아웃 후 실패 처리 |
| 기록 부족으로 생성 불가 | "기록이 충분하지 않아요. 3개 이상의 기록이 필요해요" | 최소 기록 수 체크 |

### 외부 데이터

| 상황 | 사용자에게 보이는 것 | 시스템 처리 |
|------|---------------------|------------|
| 날씨 API 실패 | 날씨 영역 숨김 또는 마지막 성공 데이터 표시 | 1시간 캐시, 실패 시 fallback |
| 급식 데이터 없음 | "등록된 급식 정보가 없습니다" | 수동 입력 유도 or 공공데이터 재조회 |
