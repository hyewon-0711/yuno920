# Yuno920 디자인 시스템 v2.0

> AI 기반 아이 성장·육아 통합 플랫폼 — 실행 가능한 디자인 시스템

---

## 디자인 원칙

| 원칙 | 설명 | 구체 기준 |
|------|------|-----------|
| 따뜻함 | 육아·성장 서비스답게 부드럽고 따뜻한 톤 | 오렌지 브랜드, 둥근 모서리, 부드러운 그림자 |
| 단순함 | 한 손으로 기록 가능, 복잡한 메뉴 최소화 | 한 화면에 1개 주요 액션, 3클릭 이내 핵심 기능 도달 |
| 일관성 | 같은 동작 = 같은 UI = 같은 위치 | 토큰 기반 스타일링, 컴포넌트 재사용 |
| 접근성 | 조부모부터 아이까지 모두 사용 가능 | WCAG AA 준수, 최소 44px 터치, 최소 14px 텍스트 |
| 모바일 우선 | 모든 화면을 375px 기준으로 먼저 설계 | 모바일 → 태블릿 → 데스크탑 순서로 확장 |

### 다크모드 결정

| 단계 | 방침 |
|------|------|
| MVP (Sprint 0~3) | **라이트 모드만** — CSS 변수 기반으로 구현하여 향후 전환 가능하게 준비 |
| 확장 (Sprint 5+) | 다크모드 추가 — 야간 기록, 수면 전 사용 대응 |

모든 컬러는 CSS 변수(토큰)로 관리하여 다크모드 전환 시 변수만 교체하는 구조로 설계합니다.

## 필수 기준

| 항목 | 기준 | 비고 |
|------|------|------|
| 색상 대비 | WCAG AA (4.5:1 일반, 3:1 대형 텍스트) | 모든 텍스트-배경 조합 확인 |

| 색상만으로 구분 금지 | 아이콘 또는 텍스트 보조 필수 | 에러=빨강 + 아이콘 + 텍스트 |

| 키보드 접근 | 포커스 링 표시 (`ring-2`) | 탭 키로 모든 인터랙션 가능 |

| 아이콘 버튼 | `aria-label` 필수 | 예: `aria-label="뒤로 가기"` |

# F. Interaction Rules (인터랙션 규칙)

---

## 탭 전환

- 하단탭 클릭 → 해당 탭 페이지로 전환 (SPA, 페이지 전환 없음)
- 전환 모션: `fade-in`, `duration-fast`
- 스크롤 위치: 탭별 독립 유지 (돌아오면 이전 위치)

## 폼 입력

- 입력 시작 → 키보드 올라옴 → 하단탭 숨김
- 입력 필드가 키보드에 가리지 않도록 자동 스크롤
- 저장 전 변경사항 있을 때 뒤로가기 → "저장하지 않고 나가시겠습니까?" 확인

## 로딩 처리

| 상황 | 처리 |
|------|------|
| 페이지 첫 로딩 | Skeleton Loader 표시 |
| 데이터 추가 로딩 | 하단 스피너 |
| 버튼 액션 (저장 등) | 버튼 Loading 상태 |
| AI 생성 (긴 대기) | 프로그레스 + "AI가 분석 중이에요" 메시지 |

## 에러 처리

| 상황 | 처리 |
|------|------|
| 네트워크 에러 | Toast (Error) + "다시 시도" 버튼 |
| 폼 유효성 에러 | 해당 필드 하단 에러 메시지 + 필드 포커스 이동 |
| 서버 에러 | 전체 화면 에러 + "새로고침" 버튼 |
| 빈 데이터 | Empty State 컴포넌트 |

## Safe Area 대응

- 상단: `env(safe-area-inset-top)` — 노치 대응
- 하단: `env(safe-area-inset-bottom)` — 홈바 대응 (하단탭에 적용)
