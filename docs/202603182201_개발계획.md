# Phase C: 기술 설계

> 목표: DB 스키마, API 설계, 프로세스 흐름을 확정

## C-1. DB 설계 (Supabase)

PRD 기반 테이블 설계를 상세화합니다:

### 테이블 목록

| 테이블 | 설명 | 관련 기능 |
|--------|------|-----------|
| `users` | 사용자 (부모) | AUTH |
| `children` | 아이 프로필 | AUTH |
| `family_members` | 가족 연결 | AUTH-03 |
| `schedules` | 일정 | HOME-01 |
| `records` | 일상 기록 | REC-01, REC-02 |
| `milestones` | 마일스톤 | REC-03 |
| `reading_logs` | 독서 기록 | REC-04, HOME-03, GROW-04 |
| `growth_metrics` | 신체/학습 지표 | GROW-01, GROW-02 |
| `hexagon_scores` | 6각형 분석 점수 | GROW-03 |
| `play_logs` | 게임 기록 | PLAY-01~04 |
| `ai_reports` | AI 리포트 | AI-01~05 |
| `notifications` | 알림 | COMM-02 |

### 관계도

```
users (1) ──── (N) children
  │                   │
  │                   ├── (N) schedules
  │                   ├── (N) records
  │                   ├── (N) milestones
  │                   ├── (N) reading_logs
  │                   ├── (N) growth_metrics
  │                   ├── (N) hexagon_scores
  │                   ├── (N) play_logs
  │                   └── (N) ai_reports
  │
  └── (N) family_members ──── (1) children
```

## C-2. API 설계

### 프론트엔드 직접 (Supabase Client)

| 영역 | 엔드포인트 패턴 | 설명 |
|------|----------------|------|
| 인증 | Supabase Auth SDK | 로그인, 회원가입, 세션 관리 |
| CRUD | Supabase Client | records, schedules, reading_logs 등 직접 접근 |
| 파일 | Supabase Storage | 사진 업로드/조회 |

### FastAPI 백엔드 (AI/외부 API)

| Method | Endpoint | 설명 | 관련 기능 |
|--------|----------|------|-----------|
| POST | `/api/ai/daily-summary` | 하루 요약 생성 | AI-01 |
| POST | `/api/ai/weekly-report` | 주간 리포트 생성 | AI-02 |
| POST | `/api/ai/monthly-report` | 월간 리포트 생성 | AI-03 |
| POST | `/api/ai/coaching` | 육아 코칭 생성 | AI-04, HOME-04 |
| POST | `/api/ai/personality` | 기질/사주 분석 | AI-05 |
| POST | `/api/ai/hexagon` | 6각형 분석 갱신 | GROW-03 |
| POST | `/api/ai/growth-advice` | 부족 영역 개선 제안 | GROW-05 |
| GET | `/api/external/weather` | 날씨 조회 | HOME-02 |
| GET | `/api/external/meal` | 급식 메뉴 조회 | HOME-02 |
| POST | `/api/play/generate-quiz` | 독서 기반 퀴즈 생성 | PLAY-03 |

## C-3. 프로세스 흐름도

### 기록 작성 프로세스

```
[사용자] 기록 작성 버튼 클릭
  → 작성 폼 표시 (텍스트 + 사진 + 카테고리)
  → [사진 있으면] Supabase Storage에 업로드
  → Supabase DB에 record 저장
  → [자동 태깅] FastAPI → OpenAI로 태그 추출
  → 태그 저장
  → 타임라인에 반영
```

### AI 리포트 생성 프로세스

```
[트리거] 사용자 요청 or 자동 스케줄 (매일 밤)
  → FastAPI가 Supabase에서 해당 기간 데이터 조회
    - records, reading_logs, play_logs, growth_metrics, schedules
  → 데이터를 구조화된 프롬프트로 조합
  → OpenAI API 호출
  → 응답을 ai_reports 테이블에 저장
  → 프론트에서 리포트 표시
```

### 게임 플레이 프로세스

```
[아이] 게임 선택
  → 난이도 결정 (이전 play_logs 기반)
  → 문제 생성 (계산: 로직 / 독서퀴즈: OpenAI)
  → 게임 플레이
  → 결과 저장 (play_logs)
  → 성장 분석 데이터 갱신 (hexagon_scores)
```
