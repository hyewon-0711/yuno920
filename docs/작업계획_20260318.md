# Yuno920 프로젝트 작업 계획

## 프로젝트 개요

| 구성 요소 | 기술 스택 |
|-----------|-----------|
| Frontend | Next.js (App Router, TypeScript, Tailwind CSS) |
| Backend (AI/데이터) | FastAPI (Python) |
| Database | Supabase |
| AI | OpenAI API |
| 배포 | Vercel (프론트엔드), Railway/Render (백엔드) |

## 목표 운영 체계

```
로컬 코드 프로젝트 생성
  → GitHub 저장소 생성/푸시
    → Vercel에 새 프로젝트로 import
      → 도메인 연결 확인
        → 이후부터 Git push 기반 운영
```

## Phase 2: 로컬 프로젝트 구조 생성

### 디렉토리 구조

```
yuno920/
├── frontend/              # Next.js 앱
│   ├── src/
│   │   ├── app/           # App Router (페이지, 레이아웃)
│   │   ├── components/    # UI 컴포넌트
│   │   ├── lib/           # Supabase 클라이언트, 유틸리티
│   │   └── styles/        # 글로벌 스타일
│   ├── public/
│   ├── package.json
│   ├── next.config.js
│   └── .env.local         # NEXT_PUBLIC_SUPABASE_URL 등
│
├── backend/               # FastAPI 앱
│   ├── app/
│   │   ├── main.py        # FastAPI 앱 진입점
│   │   ├── routers/       # API 라우트 모듈
│   │   ├── services/      # OpenAI API 호출, 비즈니스 로직
│   │   ├── models/        # Pydantic 모델
│   │   └── config.py      # 환경변수 관리
│   ├── requirements.txt
│   └── .env               # OPENAI_API_KEY, SUPABASE_KEY 등
│
├── docs/                  # 프로젝트 문서
├── .gitignore
└── README.md
```

## 이후 운영 흐름 (Git push 기반)

```
코드 수정 → git add → git commit → git push
                                        ↓
                              Vercel 자동 감지 → 빌드 → 배포
```

- **main 브랜치** push → 프로덕션 배포
- **feature 브랜치** PR → Preview 배포 (URL 자동 생성)
