## 1. 회원가입 플로우

### 트리거

사용자가 랜딩 페이지에서 "시작하기" 버튼을 클릭

### 1-A. 이메일 회원가입

```
🖥 Frontend                    🗄 Supabase                  ⚙ FastAPI
─────────────────────────────────────────────────────────────────────
│
├─ [1] 랜딩 → "시작하기" 클릭
│
├─ [2] 회원가입 폼 표시
│     - 이메일 (필수)
│     - 비밀번호 (필수, 8자 이상)
│     - 이름 (필수)
│
├─ [3] 입력값 프론트 검증
│     - 이메일 형식 확인
│     - 비밀번호 강도 확인
│     - 빈 필드 확인
│     ├─ 실패 → 에러 메시지 표시, 재입력 유도
│     └─ 성공 ↓
│
├─ [4] supabase.auth.signUp() ──────────→ [5] auth.users INSERT
│                                              │
│                                              ├─ 이메일 중복 체크
│                                              │   ├─ 중복 → error 반환
│                                              │   └─ 통과 ↓
│                                              │
│                                              ├─ [6] auth.users 레코드 생성
│                                              │
│                                              ├─ [7] 트리거: handle_new_user()
│                                              │     → public.users INSERT
│                                              │     (id, email, name)
│                                              │
│                                              └─ [8] 확인 이메일 발송 (선택)
│
├─ [9] 응답 수신 ◄──────────────────────
│     ├─ 에러 (이메일 중복)
│     │   → "이미 등록된 이메일입니다" 토스트
│     │   → 로그인 페이지 유도 링크 표시
│     │
│     └─ 성공
│         → [10] 아이 프로필 등록 화면으로 이동
│               (플로우 2로 연결)
│
```

### 1-B. 소셜 로그인 (Google / Kakao)

```
🖥 Frontend                    🗄 Supabase                  OAuth Provider
─────────────────────────────────────────────────────────────────────────
│
├─ [1] "Google로 로그인" 클릭
│
├─ [2] supabase.auth.signInWithOAuth()
│     provider: 'google'
│     redirectTo: '/auth/callback'
│     ──────────────────────────→ [3] OAuth 리다이렉트 URL 생성
│                                      │
│                                      └──────────→ [4] Google 로그인 화면
│                                                       │
│                                                       ├─ 사용자 인증
│                                                       │   ├─ 취소 → 랜딩으로 복귀
│                                                       │   └─ 승인 ↓
│                                                       │
│ ◄────────────────── [5] /auth/callback?code=xxx ◄─────┘
│
├─ [6] callback 페이지에서 세션 교환
│     supabase.auth.exchangeCodeForSession()
│     ──────────────────────────→ [7] code → JWT 교환
│                                      │
│                                      ├─ 최초 가입 → auth.users 생성
│                                      │   → 트리거: public.users 생성
│                                      │   → children 존재 여부 확인
│                                      │     ├─ 없음 → 아이 프로필 등록으로
│                                      │     └─ 있음 → Home으로
│                                      │
│                                      └─ 기존 사용자 → 세션 반환
│
├─ [8] 응답 수신 ◄──────────────────────
│     ├─ 최초 가입 → 아이 프로필 등록 (플로우 2)
│     └─ 기존 사용자 → Home Dashboard
│
```

### 에러 처리

| 단계 | 에러 | 처리 |
|------|------|------|
| 3 | 입력값 검증 실패 | 해당 필드 빨간 테두리 + 인라인 에러 메시지 |
| 5 | 이메일 중복 | "이미 등록된 이메일입니다" + 로그인 링크 |
| 5 | 네트워크 오류 | "네트워크 오류가 발생했습니다. 다시 시도해주세요" |
| 4 (소셜) | OAuth 실패 | "로그인에 실패했습니다. 다시 시도해주세요" |

### 성공 결과

- `auth.users` + `public.users` 레코드 생성
- JWT 세션 토큰 발급
- 아이 프로필 등록 화면으로 이동

## 2. 아이 프로필 등록 플로우

### 트리거

회원가입 완료 후 자동 진입, 또는 설정 화면에서 "+ 아이 추가" 클릭

```
🖥 Frontend                    🗄 Supabase
──────────────────────────────────────────────
│
├─ [1] 아이 프로필 등록 폼 표시
│     - 아바타 사진 (선택)
│     - 이름 (필수)
│     - 생년월일 (필수, DatePicker)
│     - 성별 (필수, 라디오)
│     - 출생 시간 (선택, TimePicker)
│
├─ [2] 사진 선택 시 (선택)
│     ├─ 이미지 선택/촬영
│     ├─ 이미지 리사이즈 (512x512)
│     ├─ supabase.storage.upload() ────→ [3] avatars 버킷 저장
│     │                                       │
│     ├─ URL 수신 ◄────────────────────────────┘
│     └─ 폼에 avatar_url 설정
│
├─ [4] "시작하기" 클릭
│     ├─ 프론트 검증
│     │   - 이름 빈 값 → "이름을 입력해주세요"
│     │   - 생년월일 미래 → "올바른 날짜를 선택해주세요"
│     │   - 성별 미선택 → "성별을 선택해주세요"
│     │   ├─ 실패 → 에러 표시
│     │   └─ 통과 ↓
│     │
│     └─ supabase.from('children').insert() ──→ [5] children 테이블 INSERT
│                                                    │
│                                                    ├─ [6] 트리거: handle_new_child()
│                                                    │     → family_members INSERT
│                                                    │       (user_id, child_id, role='admin')
│                                                    │
│                                                    └─ 응답 반환
│
├─ [7] 응답 수신 ◄────────────────────────────
│     ├─ 에러 → "프로필 등록에 실패했습니다" 토스트
│     │         재시도 유도
│     │
│     └─ 성공
│         ├─ 토스트: "[이름]의 프로필이 등록되었습니다"
│         └─ Home Dashboard로 이동
│
```

### 데이터 변환

| 입력 | 변환 | 저장값 |
|------|------|--------|
| 이름 | trim() | "윤호" |
| 생년월일 | DatePicker → ISO | "2018-09-20" |
| 성별 | 라디오 → enum | "male" |
| 출생 시간 | TimePicker → time | "14:30:00" 또는 null |
| 사진 | Storage upload → URL | "https://...supabase.co/storage/v1/..." |

## 3. 기록 작성 플로우

### 트리거

- Home "오늘 기록하기" FAB 클릭
- Record 탭 "+" 버튼 클릭

```
🖥 Frontend                    🗄 Supabase                  ⚙ FastAPI         🤖 OpenAI
──────────────────────────────────────────────────────────────────────────────────────────
│
├─ [1] 기록 입력 방식 선택
│     ├─ Quick Record (바텀시트)
│     └─ Full Record (전체 화면)
│
│ === Quick Record 경로 ===
├─ [2a] 바텀시트 표시
│     - 기분 선택 (😊😐😢🤒😴)
│     - 한 줄 메모 (텍스트 입력)
│
├─ [3a] "저장" 클릭
│     └─ supabase.from('records').insert({
│          child_id, content, mood,
│          categories: [], recorded_at: today
│        }) ──────────────────────────→ [4a] records INSERT
│                                            │
│ ◄──────────────────────────────────────────┘
│     ├─ 성공 → 토스트 + Home 복귀
│     └─ 실패 → "저장에 실패했습니다" 토스트
│
│ === Full Record 경로 ===
├─ [2b] 기록 작성 화면 표시
│     - 날짜 선택 (기본: 오늘)
│     - 기분 선택
│     - 카테고리 복수 선택
│     - 텍스트 입력 (필수)
│     - 사진 첨부 (최대 5장)
│
├─ [3b] 사진 첨부 시 (선택)
│     ├─ 이미지 선택 (갤러리 또는 카메라)
│     ├─ 이미지 압축 (max 1MB)
│     ├─ 각 사진에 대해:
│     │   supabase.storage.upload() ──→ [4b] record-photos 버킷 저장
│     │                                       │
│     │   URL 수신 ◄──────────────────────────┘
│     │     ├─ 성공 → photos 배열에 URL 추가
│     │     └─ 실패 → "사진 업로드 실패" (해당 사진 제외)
│     └─ 모든 사진 업로드 완료 (또는 부분 실패)
│
├─ [5b] "저장" 클릭
│     ├─ 프론트 검증
│     │   - content 빈 값 체크
│     │   ├─ 실패 → "내용을 입력해주세요"
│     │   └─ 통과 ↓
│     │
│     └─ supabase.from('records').insert({
│          child_id, user_id, content,
│          mood, categories, photos,
│          recorded_at
│        }) ──────────────────────────→ [6b] records INSERT
│                                            │
│ ◄──────────────────────────────────────────┘
│     ├─ 실패 → "저장에 실패했습니다" 토스트
│     └─ 성공 ↓
│
├─ [7b] 자동 태깅 요청 (비동기, 사용자 대기 없음)
│     POST /api/ai/auto-tag ─────────────────→ [8b] content 분석
│       { content, mood }                            │
│                                                    └──→ [9b] OpenAI API
│                                                          프롬프트:
│                                                          "다음 아이 일상 기록에서
│                                                           카테고리와 키워드를
│                                                           추출하세요: {content}"
│                                                          │
│                                                   ◄──────┘
│                                              [10b] 결과 파싱
│                                              { categories, keywords,
│                                                is_milestone }
│                                                    │
│ ◄──────────────────────────────────────────────────┘
│
├─ [11b] 태그 결과 반영 (백그라운드)
│     supabase.from('records')
│       .update({ categories: AI결과 })
│       .eq('id', recordId) ─────────→ [12b] records UPDATE
│
├─ [13b] 토스트: "기록이 저장되었습니다 ✅"
│     └─ Record 탭 타임라인으로 이동
│
```

### 에러 처리

| 단계 | 에러 | 처리 |
|------|------|------|
| 4b | Storage 업로드 실패 | 해당 사진 제외, 나머지로 진행. "일부 사진 업로드 실패" 토스트 |
| 6b | DB INSERT 실패 | "저장에 실패했습니다" 토스트, 재시도 유도 |
| 9b | OpenAI 호출 실패 | 태그 없이 기록 유지 (categories: []), 백그라운드 재시도 큐 등록 |
| 9b | 응답 파싱 실패 | 태그 없이 기록 유지 |

## 4. 독서 기록 플로우

### 트리거

- Record 탭 → "+" → "독서 기록" 선택
- Home → 독서 현황 카드 → "+ 독서 기록" (P1)

```
🖥 Frontend                    🗄 Supabase
──────────────────────────────────────────────
│
├─ [1] 독서 기록 작성 폼 표시
│     - 책 제목 (필수, 텍스트)
│     - 카테고리 (필수, 단일 선택)
│       [과학][동화][역사][예술][수학][사회][기타]
│     - 읽은 날짜 (기본: 오늘)
│     - 읽은 시간 (필수, 분 단위 슬라이더 또는 입력)
│     - 메모 (선택, 텍스트)
│
├─ [2] "저장" 클릭
│     ├─ 프론트 검증
│     │   - 제목 빈 값 → "책 제목을 입력해주세요"
│     │   - 시간 0 또는 미입력 → "읽은 시간을 입력해주세요"
│     │   ├─ 실패 → 에러 표시
│     │   └─ 통과 ↓
│     │
│     └─ supabase.from('reading_logs').insert({
│          child_id, title, category,
│          duration_minutes, memo, read_date
│        }) ──────────────────────────→ [3] reading_logs INSERT
│                                            │
│ ◄──────────────────────────────────────────┘
│     ├─ 실패 → "저장에 실패했습니다" 토스트
│     └─ 성공 ↓
│
├─ [4] 관련 UI 갱신
│     ├─ Record 탭 타임라인: 📚 아이콘으로 독서 기록 표시
│     ├─ Home 독서 현황: 오늘 독서 시간 + 목표 달성률 갱신 (P1)
│     └─ Growth 독서 성장: 월별 통계 반영 (P1)
│
├─ [5] 토스트: "독서 기록이 저장되었습니다 📚"
│
```

### 데이터 변환

| 입력 | 변환 | 저장값 |
|------|------|--------|
| 책 제목 | trim() | "어린왕자" |
| 카테고리 | 버튼 → enum | "fairy_tale" |
| 읽은 날짜 | DatePicker → ISO | "2026-03-18" |
| 읽은 시간 | 숫자 입력 → int | 30 (분) |
| 메모 | trim() or null | "3장까지 읽음" |

```
│     ├─ 실패 → "저장에 실패했습니다"
│     └─ 성공 ↓
│
├─ [4] Home 일정 목록 갱신
│     └─ 토스트: "일정이 추가되었습니다"
│
│ === 일정 수정 ===

├─ [6] "저장" 클릭

│ ◄──────────────────────────────────────────┘
│     └─ 목록 갱신 + 토스트

│ === 일정 삭제 ===

│         supabase.from('schedules').delete()

│                                          │
│ ◄────────────────────────────────────────┘
│     └─ 목록 갱신 + 토스트: "일정이 삭제되었습니다"
│
```

## 6. 성장 데이터 입력 플로우

### 트리거

Growth 탭 → 신체 성장 카드 → "+ 입력" 클릭

```
🖥 Frontend                    🗄 Supabase
──────────────────────────────────────────────
│
├─ [1] 성장 데이터 입력 모달 표시
│     - 측정 날짜 (기본: 오늘)
│     - 키 (cm, 소수점 1자리)
│     - 몸무게 (kg, 소수점 1자리)
│     - SR 점수 (0~100, 정수, 선택)
│
├─ [2] "저장" 클릭
│     ├─ 검증
│     │   - 키 또는 몸무게 중 최소 1개 필수
│     │   - 키: 50~250 범위
│     │   - 몸무게: 5~150 범위
│     │   - SR 점수: 0~100 범위
│     │   ├─ 실패 → 에러 표시
│     │   └─ 통과 ↓
│     │
│     └─ supabase.from('growth_metrics').insert({
│          child_id, height, weight,
│          sr_score, recorded_at
│        }) ──────────────────────────→ [3] growth_metrics INSERT
│                                            │
│ ◄──────────────────────────────────────────┘
│     ├─ 실패 → "저장에 실패했습니다"
│     └─ 성공 ↓
│
├─ [4] Growth 화면 갱신
│     ├─ 신체 성장 그래프: 새 데이터 포인트 추가
│     │   (차트 라이브러리로 실시간 렌더링)
│     ├─ 최신 키/몸무게 수치 업데이트

│
├─ [5] 토스트: "성장 데이터가 기록되었습니다 📏"
│
```

## 12. 6각형 점수 갱신 플로우

### 트리거

- 게임 플레이 완료 시 (플로우 7에서 호출)
- 기록 저장 시 (비동기)
- Insight 탭에서 수동 갱신 요청

```
🖥 Frontend                    ⚙ FastAPI                    🗄 Supabase         🤖 OpenAI
──────────────────────────────────────────────────────────────────────────────────────────
│
├─ [1] 갱신 트리거 발생
│     (게임 완료 / 기록 저장 / 수동 요청)
│
├─ [2] 6각형 점수 산출 요청
│     POST /api/ai/hexagon ─────────────→ [3] 데이터 수집 (최근 30일)
│       { child_id, period_days: 30 }          │
│                                              ├─ play_logs 조회 (30일)
│                                              │   ├─ math 게임: 점수, 정답률
│                                              │   ├─ memory 게임: 레벨, 시간
│                                              │   └─ reading_quiz: 정답률
│                                              │
│                                              ├─ records 조회 (30일)
│                                              │   ├─ 카테고리별 기록 수
│                                              │   └─ 감정 분포
│                                              │
│                                              ├─ reading_logs 조회 (30일)
│                                              │   ├─ 총 독서 시간
│                                              │   ├─ 독서 권 수
│                                              │   └─ 분야 다양성
│                                              │
│                                              ├─ growth_metrics 조회 (최신)
│                                              │
│                                              └─ [4] 점수 산출 로직
│                                                   │
│                                                   ├─ 학습 (learning)
│                                                   │   = math_score × 0.3
│                                                   │   + reading_quiz_accuracy × 0.3
│                                                   │   + sr_score_latest × 0.2
│                                                   │   + reading_diversity × 0.2
│                                                   │
│                                                   ├─ 신체 (physical)
│                                                   │   = physical_records_count × 10
│                                                   │   + growth_regularity × 50
│                                                   │   (cap: 100)
│                                                   │
│                                                   ├─ 사회성 (social)
│                                                   │   = social_records_count × 15
│                                                   │   + 기본값 50
│                                                   │   (cap: 100)
│                                                   │
│                                                   ├─ 감정 (emotion)
│                                                   │   = mood_variety × 0.3
│                                                   │   + positive_ratio × 0.4
│                                                   │   + emotion_records × 0.3
│                                                   │
│                                                   ├─ 창의성 (creativity)
│                                                   │   = reading_variety × 0.3
│                                                   │   + creative_records × 0.4
│                                                   │   + memory_game_level × 0.3
│                                                   │
│                                                   └─ 습관 (habit)
│                                                       = daily_record_streak × 0.4
│                                                       + reading_consistency × 0.3
│                                                       + schedule_adherence × 0.3
│                                                   │
│                                                   │
│                                              ◄────┘
│                                              │
│                                              ├─ [5] AI 코멘트 생성 (선택)
│                                              │     OpenAI에 점수 + 변화량 전달
│                                              │     → 한 줄 코멘트 생성
│                                              │     "창의성 영역이 빠르게
│                                              │      성장하고 있어요 🎨"
│                                              │
│                                              ├─ [6] hexagon_scores INSERT
│                                              │     { child_id, learning, physical,
│                                              │       social, emotion, creativity,
│                                              │       habit, calculated_at }
│                                              │     ──────────────→ 🗄 INSERT
│                                              │
│                                              └─ 응답 반환
│
│ ◄──────────────────────────────────────────
│     ├─ 실패 → 기존 점수 유지, 에러 로그
│     └─ 성공 ↓
│
├─ [7] Growth 화면 갱신 (현재 화면이 Growth인 경우)
│     ├─ 레이더 차트 업데이트
│     ├─ AI 코멘트 표시
│     └─ 이전 대비 변화량 표시
│         (↑+5, ↓-1 등)
│
```

### 점수 산출 세부 기준

| 영역 | 데이터 소스 | 가중치 요소 |
|------|------------|------------|
| **학습** | play_logs(math), play_logs(reading_quiz), growth_metrics(sr_score), reading_logs | 게임 점수, 퀴즈 정답률, SR 점수, 독서 다양성 |
| **신체** | records(categories='health'/'play'), growth_metrics | 신체 관련 기록 빈도, 성장 데이터 입력 규칙성 |
| **사회성** | records(categories='emotion'/'social') | 사회성 관련 기록 빈도, 활동 다양성 |
| **감정** | records(mood), records(categories='emotion') | 감정 기록 다양성, 긍정 비율, 감정 표현 빈도 |
| **창의성** | reading_logs(category 다양성), play_logs(memory), records | 독서 분야 다양성, 기억력 게임 레벨, 창의적 활동 기록 |
| **습관** | 일별 기록 연속성, reading_logs 규칙성, schedules 이행률 | 매일 기록 streak, 독서 꾸준함, 일정 관리 |
